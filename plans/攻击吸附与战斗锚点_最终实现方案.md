# 攻击吸附 + 战斗锚点 — 最终实现方案与代码框架

本文档汇总当前已讨论定案的内容，作为**唯一实现蓝图**：  
**锚点配表嵌在 `Actor` 内（不新增组件）**；**吸附逻辑由非 Mono 的 Session 编排**；**剑身中心 ↔ 敌人胶囊表面间隙由静态几何类计算**；**Timeline 用 `ExposedReference` 可选覆盖锚点**。

---

## 1. 目标与原则

| 原则 | 说明 |
|------|------|
| Prefab 不堆新脚本 | 战斗锚点以 `Actor` 上 `[SerializeField]` 列表存在，不增加 `ActorCombatAnchors` 组件 |
| Timeline 薄 | `ActionMagnetismV2Behavior` 只做生命周期 + 解析 `ExposedReference` + 调 Session |
| 状态进 Session | 每段 Clip 一个 `ActionMagnetismSession` 实例（非 Mono），持有 `_didInstantMove`、缓存方向等 |
| 几何无状态 | `MagnetismCapsuleGeometry` 静态纯函数，不引用 Actor |
| 锚点解析顺序 | `ExposedReference` 解析成功且非 null → 否则 `actor.TryGetCombatAnchor(id)` → 否则表面修正跳过并可选 Log |

---

## 2. 架构总览

```
Timeline ActionMagnetismV2Clip
    ├─ MagnetismConfig（Serializable）
    └─ ExposedReference<Transform> weaponAnchorOverride（可选）

ActionMagnetismV2Behavior (PlayableBehaviour)
    OnClipStart  → new ActionMagnetismSession(...); Begin()
    OnClipUpdate → session.Tick(delta)
    OnClipStop   → session.End()

ActionMagnetismSession (plain C# class)
    Tick: 水平距离/maxDistance → 粗 desiredPos(attachDistance)
        → [可选] MagnetismCapsuleGeometry 修正根位置（剑身中心 vs 敌人胶囊 + surfaceClearance）
        → ApplyMove / ApplyRotation

Actor
    ├─ List<CombatAnchorEntry> + Dictionary 缓存
    └─ TryGetCombatAnchor(CombatAnchorId, out Transform)

MagnetismCapsuleGeometry (static)
    TryGetClearanceInfo / ComputeRootDeltaForClearance（实现阶段填充）
```

## 4. 代码框架（签名级，实现时填空）

### 4.1 `CombatAnchors.cs`（`CombatAnchorId` + `CombatAnchorEntry` 同文件）

```csharp
public enum CombatAnchorId
{
    WeaponBladeMid,
    RightHand,
    LeftHand,
}

[System.Serializable]
public struct CombatAnchorEntry
{
    public CombatAnchorId id;
    public Transform transform; // 可空
}
```

### 4.2 `Actor.cs` 增量

```csharp
// 字段
[SerializeField] private List<CombatAnchorEntry> _combatAnchors;
private Dictionary<CombatAnchorId, Transform> _combatAnchorCache;

// Awake 末尾：BuildCombatAnchorCache() — 遍历 _combatAnchors，重复 id 打 Warning，跳过 null transform

public bool TryGetCombatAnchor(CombatAnchorId id, out Transform t)
{
    return _combatAnchorCache != null && _combatAnchorCache.TryGetValue(id, out t) && t != null;
}
```

### 4.3 `MagnetismConfig` 扩展字段

在现有字段基础上增加：

```csharp
[Header("Weapon anchor (default: Actor combat anchor table)")]
public CombatAnchorId weaponAnchorId = CombatAnchorId.WeaponBladeMid;

[Header("Surface clearance (weapon center vs enemy capsule)")]
public bool useWeaponSurfaceClearance = false;

public float surfaceClearance = 0.05f;
[Tooltip("每帧根节点沿法线最大修正量（米，0=不限制）")]
```

### 4.4 `MagnetismCapsuleGeometry.cs`（静态）

```csharp
public static class MagnetismCapsuleGeometry
{
    /// <summary>剑身中心到胶囊表面：最近点、朝外法线、带符号距离（正=外，负=穿入）。</summary>
    public static bool TryGetClearanceInfo(
        Vector3 weaponCenterWorld,
        CapsuleCollider enemyCapsule,
        out Vector3 closestSurfacePoint,
        out Vector3 outwardNormal,
        out float signedDistance);

    /// <summary>在水平面或全空间上，把 actor 根位移一笔修正，使间隙趋向 targetClearance（实现可迭代 1～2 次）。</summary>
    public static Vector3 ComputeRootDeltaForClearance(
        Vector3 actorRootWorld,
        Vector3 weaponCenterWorld,
        CapsuleCollider enemyCapsule,
        float targetClearance,
        bool horizontalOnly,
        float maxDeltaMagnitude);
}
```

### 4.5 `ActionMagnetismSession.cs`（非 Mono）

```csharp
public sealed class ActionMagnetismSession
{
    public ActionMagnetismSession(
        Actor actor,
        Transform combatTarget,
        MagnetismConfig config,
        Transform weaponCenterTransform); // 已由 Behaviour 解析好（override 或表或 null）

    public void Begin();
    public void Tick(float deltaTime);
    public void End();

    // 私有：ApplyMoveInstant/Towards、ApplyRotation、取敌人 CapsuleCollider 等
}
```

**Tick 内顺序（固定）**：

1. 校验 `actor`、`combatTarget`、`config`  
2. `toTarget` 水平化、`maxDistance` 早退  
3. 更新 `_cachedHorizontalDir`  
4. `desiredPos` = 方案一 `attachDistance`（与当前 V2 行为一致：已贴身则 `desiredPos = actor.position`）  
5. 若 `useWeaponSurfaceClearance && weaponCenterTransform && enemyCapsule`：  
   `delta = MagnetismCapsuleGeometry.ComputeRootDeltaForClearance(...)`，`desiredPos += delta`（或迭代采样，实现时定）  
6. `InstantMove` / `SpeedMove` 应用位置  
7. 旋转：`ActorMovement` `SetRotationSpeedOverride` / `SetRotationInstant` / `UpdateRotation`  
8. `End()`：`SetRotationSpeedOverride(-1)`

### 4.7 `ActionMagnetismV2Behavior.cs`（变薄）

```csharp
public ExposedReference<Transform> weaponAnchorOverride;

private ActionMagnetismSession _session;
private PlayableDirector _director; // OnPlayableCreate 或 OnGraphStart 取 graph.GetResolver()

protected override void OnClipStart(Playable playable)
{
    // director = playable.GetGraph().GetResolver() as PlayableDirector;
    Transform weapon = null;
    var go = weaponAnchorOverride.Resolve(director);
    if (go != null) weapon = go;
    else if (actor.TryGetCombatAnchor(config.weaponAnchorId, out var t)) weapon = t;

    _session = new ActionMagnetismSession(actor, target, config, weapon);
    _session.Begin();
}

protected override void OnClipUpdate(Playable playable, FrameData info)
{
    _session?.Tick((float)info.deltaTime);
}

protected override void OnClipStop(bool isNormal)
{
    _session?.End();
    _session = null;
}
```

**注意**：`ExposedReference.Resolve` 需要 `PlayableDirector`；若 `ActionBehaviourBase.OnGraphStart` 未缓存 director，可在 Behaviour 里 `override OnGraphStart` 存 `_director`，或在 `OnClipStart` 用 `playable.GetGraph().GetResolver()`。

## 5. 敌人受击体（胶囊）怎么拿

### 5.1 第一版（实现快）

- 在 `CombatTarget` 根物体上：`GetComponent<CapsuleCollider>()`（或 `GetComponentInChildren`，若胶囊在子物体）。
- **若没有**：`useWeaponSurfaceClearance` 时跳过表面间隙修正，只保留根节点 `attachDistance` 吸附；`debugLog` 可打一条说明。

### 5.2 第二版是什么意思（可选演进）

不是换引擎 API，而是**换“找 Collider 的方式”**，避免写死 `GetComponent`、敌人结构一变就挂：

- 敌人身上挂一个小组件，例如 `CombatHurtbox` / 实现接口 `IHurtbox`，**明确暴露**「用于几何/受击的 `CapsuleCollider`」（或 `Collider` 列表里标记主受击体）。
- 吸附 Session 只调 `hurtbox.MainCapsule`，不猜层级。

**何时值得上第二版**：敌人多个胶囊、根上无 Collider、或程序/策划需要换受击体而不改吸附代码时。

## 6. 如何「精准」控制玩家攻击与敌人的合适距离

「精准」在这里指：**可控、可重复、对齐你关心的视觉参考点（剑身中心）与敌人几何**，而不是单一魔法数字。

### 6.1 两层距离（一起用）

| 层级 | 参数 | 控制什么 |
|------|------|----------|
| **粗调（根节点）** | `attachDistance`、`maxDistance`、`approachMode` / `approachSpeed` | 玩家 **Actor 根**在水平面上相对敌人 pivot（或目标点）的大致站位；超出 `maxDistance` 整段不吸。 |
| **细调（视觉接触）** | `useWeaponSurfaceClearance`、`surfaceClearance`、`weaponAnchorId` / Exposed 覆盖、`MagnetismCapsuleGeometry` | 在粗站位之后，用 **剑身中心世界坐标** 相对 **敌人胶囊表面** 再推/拉 **根节点**，使间隙 ≈ `surfaceClearance`，减少穿模。 |

### 6.2 单帧里在做什么（概念）

1. 用水平向量算「朝向敌人的方向」和「根节点粗目标点」（`attachDistance` 方案一）。  
2. 若开启表面修正：取武器锚点 `Transform.position` 作为剑身中心采样点；取敌人胶囊；算「当前间隙 vs 目标 `surfaceClearance`」差多少，得到 **根位移修正量**（可限制 `maxRootCorrectionPerFrame` 防一帧过猛）。  
3. 应用位移 + 旋转，使下一帧动画下剑身中心仍趋向合格间隙（必要时 1～2 次迭代，实现阶段定）。

### 6.3 仍依赖哪些「美术/配置」

- **锚点**：Prefab 里 `WeaponBladeMid` 是否真在剑身中心。  
- **敌人胶囊**：是否大致包住可视身体；太小/太偏会让几何修正和肉眼不一致。  
- **Timeline**：个别招式可用 `ExposedReference` 换锚点，避免一刀一刀切枚举。

### 6.4 和「只调碰撞体大小」的区别

只调胶囊半径是**间接**改间隙；本方案是**直接**以「剑身中心到表面距离」为目标推根，更贴你要的「划过身体」观感，且对**不同体型敌人**更稳（同一 `surfaceClearance` 语义不变）。

## 7. Prefab / Timeline 工作流

1. **Actor Prefab**：在 `Actor` 组件上展开「Combat Anchors」，为 `WeaponBladeMid` 拖剑身中心 Transform。

3. **特例招式**：在 Clip 上绑定 `ExposedReference` 覆盖。
