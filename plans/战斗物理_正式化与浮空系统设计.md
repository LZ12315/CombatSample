# 战斗物理：试错脚本正式化 + 浮空系统设计备忘

## 一、如何把现在的试错脚本变成「正式功能」

### 1.1 模块定位（已经做的）

| 脚本 | 命名空间 | 职责 |
|------|-----------|------|
| `CharacterControllerRigidbodyPush` | `CombatSample.PhysicsInteraction` | 玩家 CC 对**指定 Layer** 上的动态刚体施加水平挤开 |
| `RigidbodyHorizontalDamping` | `CombatSample.PhysicsInteraction` | 可推动体水平滑停，与推挤配对 |

原则：**不塞进 `ActorMovement` 内部**，保持「移动汇总」与「对外物理交互」分离，以后换 KCC 只换移动核，交互接口可复用思路。

### 1.2 配置规范（建议在项目里固定下来）

1. **Layer**  
   - 可推动小怪：`Enemy`（或 `Pushable`）  
   - 环境静态阻挡：`Default` / `Obstacle`  
   - 玩家 `CharacterControllerRigidbodyPush` 上 **Pushable Layers 只勾可推层**，避免误推箱子/机关（除非你刻意要推）。

2. **Prefab 分级**  
   - **可推动木桩**：`Rigidbody` Dynamic + 非 Trigger 身体 Collider + `RigidbodyHorizontalDamping` + 适当 `Drag`（可选）  
   - **真木桩**：`Rigidbody.isKinematic = true`，不挂阻尼脚本  

3. **数值归属**  
   - 推挤强度：以 **玩家侧** `CharacterControllerRigidbodyPush` 为唯一真相（按角色/难度可做 **Prefab 变体**，不必上 ScriptableObject 除非多职业共用一套表）。  
   - 滑停速度：以 **敌人 Prefab** 上 `RigidbodyHorizontalDamping` 为真相（胖怪阻尼可略低、瘦怪略高）。

4. **验收清单**  
   - [ ] 走路轻擦不飞、跑步/攻击根运动顶人明显  
   - [ ] 松开后敌人在 0.x～1s 内停住（可调阻尼）  
   - [ ] 不误推不该推的 Layer  
   - [ ] 与 Magnetism / Timeline 位移同时存在时无双倍异常（必要时在招式里 **Disable** 推挤组件）

### 1.3 可选下一步（需要时再写代码）

- **`ScriptableObject` 配置表**：`PhysicsInteractionProfile`（pushGain、maxImpulse、damping），多角色引用同一资产。  
- **接口 `IPushReaction`**：阻尼脚本改为对实现接口的物体调 `OnPushed(Vector3 delta)`，便于以后接受击硬直。  
- **调试 Gizmo**：仅在 Development Build 画本帧 `moveLength` 方向。

## 二、更复杂的「浮空系统」该怎么设计（面向现有 ActorMovement + Animancer）

核心矛盾：**谁在某一帧拥有「竖直速度 / 重力 / 根运动 Y」的权威**。浮空一复杂，必须先把权威写清楚。

### 2.1 建议的分层（概念）

```
┌─────────────────────────────────────┐
│  Locomotion / ActionState（状态机）  │  是否在地面、是否允许二段跳、是否锁输入
├─────────────────────────────────────┤
│  AirborneMotor（浮空子系统，未来抽） │  重力曲线、最大下落速度、空闪、吹飞
├─────────────────────────────────────┤
│  ActorMovement                       │  汇总本帧位移，唯一调用 CC.Move
└─────────────────────────────────────┘
```

- **地面**：保持现状，根运动 + `CodeDriven` 以水平为主，`PerformGravity` 管贴地。  
- **浮空**：不要到处 `transform.position +=`；统一变成「本帧给 `ActorMovement` 一个 **额外竖直/全向速度** 或 **叠加增量**」，仍由 **一次 `Move`** 执行（与推挤、吸附一致）。

### 2.2 `MovementMode` 扩展思路（与现有枚举对齐）

在 `ActorMovement` 上预留（实现可分期）：

| 模式 | 含义 |
|------|------|
| `RootMotion` | 现行地面为主 |
| `CodeDriven` | 现行 |
| `MotionWarping` | 吸附、技能位移等「覆盖/混合」 |
| **`Airborne`（未来）** | 重力 + 空中控制 + 与根运动混合规则（例如：仅采用根运动水平、竖直用代码） |

关键规则用**数据**写清，例如：

- **Knock-up**：短时间 `Airborne`，初速度 `+Y`，水平阻尼或硬直由受击系统给。  
- **Air combo**：`Airborne` + 允许水平输入缩放系数 `airControl`。  
- **落地**：射线/CharacterController.isGrounded 切回 `RootMotion`，**清掉**累积空中速度，避免落地滑。

### 2.3 与 Animancer / 根运动的关系

- **方案 A（常见）**：空中**禁用**动画根运动竖直分量，只信代码重力；水平可混根运动（连招位移）或完全代码。  
- **方案 B**：空中全根运动（难调，易和物理打架），仅适合强演出段并用 Timeline 控制。

建议默认 **方案 A**，演出关用 Timeline **临时**切模式。

### 2.4 与 Timeline / Action 的关系

- 浮空段尽量仍是：**Action 或状态** 设置 `Airborne` + 初速度，**不在**每个 Playable 里直接改 Transform。  
- HitStop、浮空顿帧：注意 **FixedUpdate vs Update** 与 `Time.timeScale`；冲量建议在 `FixedUpdate` 或统一在 `ActorMovement` 里按 `deltaTime` 累积。

### 2.5 不要一开始就上的东西

- 不要为了浮空立刻换 KCC；先把 **「地面 / 空中权威」** 和 **`ActorMovement` 单入口** 写死，再决定 motor 实现。  
- 多段跳、空闪：做成 **能力 Tag + 计数器**，由状态机重置（落地清零），避免散落在输入脚本里。
