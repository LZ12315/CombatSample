通用修改指南

1. 类名更改

如果业务代码中的类名更改：
• InputCommand → 修改 InputCommandEditor.cs 中的 [CustomEditor(typeof(InputCommand))]

• InputDataCheck → 修改 InputDataCheckDrawer.cs 中的 [CustomPropertyDrawer(typeof(InputDataCheck))]

• ButtonCheckSetting → 修改 ButtonCheckSettingDrawer.cs 中的 [CustomPropertyDrawer(typeof(ButtonCheckSetting))]

• JoystickCheckSetting → 修改 JoystickCheckSettingDrawer.cs 中的 [CustomPropertyDrawer(typeof(JoystickCheckSetting))]

2. 字段名更改

如果业务代码中的字段名更改：
• 在对应的 PropertyDrawer 中修改 FindProperty 或 FindPropertyRelative 中的字符串

• 确保所有引用该字段的地方都更新

3. 添加新字段

3. 在 GetPropertyHeight 方法中添加高度计算：
   height += EditorGUI.GetPropertyHeight(newFieldProp, true);

最佳实践

1. 使用常量：
   // 在类顶部定义常量
   private const string WaitTimePropName = "waitTime";
   private const string DataChecksPropName = "dataChecks";
   
   // 使用时
   waitTimeProp = serializedObject.FindProperty(WaitTimePropName);
   
   这样当字段名更改时，只需修改常量值

2. 添加注释：
   // 对应业务代码中的 waitTime 字段
   waitTimeProp = serializedObject.FindProperty("waitTime");
   
   这样便于理解字段与业务代码的对应关系

3. 使用搜索替换：
   • 当字段名更改时，使用编辑器的全局搜索替换功能

   • 确保替换所有相关文件中的引用

4. 版本控制：
   • 在修改前提交代码到版本控制系统

   • 这样如果修改出错可以轻松回滚
